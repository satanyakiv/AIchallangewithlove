AI Challenge — 35-денний курс з побудови AI-агентів

Про що цей курс
────────────────
Практичний курс, що веде від першого API-виклику до повноцінних AI-систем.
За 5 тижнів учасник створює агента з пам'яттю, станами, RAG-пошуком,
MCP-інтеграціями і локальним LLM-бекендом.

Кожен день — окреме завдання з конкретним результатом (код + відео).
Курс побудований від простого до складного: спочатку основи LLM API,
потім архітектура агентів, далі — інтеграція з зовнішніми системами.


Навіщо це сучасному інженеру
────────────────────────────
- AI-агенти стають стандартним компонентом продуктів: чат-боти, асистенти,
  автоматизація workflow, code review, підтримка користувачів
- Знання RAG, MCP, prompt engineering, управління контекстом — must-have
  для backend/fullstack/mobile інженерів, що працюють з LLM
- Розуміння обмежень моделей (токени, галюцинації, контекстне вікно)
  критичне для побудови надійних AI-систем
- Локальні LLM — зростаючий тренд для приватності та зниження витрат


Технології
──────────
LLM API          DeepSeek (chat, reasoner), сумісні з OpenAI API
Фреймворк        Kotlin, Ktor, Compose Multiplatform (KMP)
AI SDK            Koog (Kotlin AI agent framework)
Інтеграція        MCP (Model Context Protocol) — стандарт для tool use
RAG               Embeddings, chunking, FAISS/SQLite, reranking
Локальні LLM     Ollama, LM Studio, llama.cpp, Mistral, Qwen, Phi
DI                Koin
Тестування        MockK, Ktor test-host
Бази даних        Room + SQLite (KMP), in-memory stores


Структура курсу
───────────────

Тиждень 1 (Дні 1-5): Основи LLM API
  День 1-3   Вступ, налаштування, перший запит (відсутні в репо)
  День 4     Температура — вплив параметрів на відповіді
  День 5     Версії моделей — порівняння слабких/середніх/сильних моделей

Тиждень 2 (Дні 6-10): Перший агент + управління контекстом
  День 6     Перший агент — інкапсульована логіка запитів до LLM
  День 7     Збереження контексту — JSON/SQLite, відновлення після рестарту
  День 8     Робота з токенами — підрахунок, ліміти, вартість
  День 9     Стиснення історії — summary замість повної історії
  День 10    Стратегії контексту — Sliding Window, Sticky Facts, Branching

Тиждень 3 (Дні 11-15): Пам'ять, стани, інваріанти
  День 11    Модель пам'яті — короткострокова / робоча / довготривала
  День 12    Персоналізація — профіль користувача, адаптація відповідей
  День 13    State Machine — формалізований стан задачі
  День 14    Інваріанти — обмеження, які агент не має права порушувати
  День 15    Контрольовані переходи — заборона "перестрибування" етапів

Тиждень 4 (Дні 16-20): MCP (Model Context Protocol)
  День 16    Підключення MCP — SDK/клієнт, список інструментів
  День 17    Перший MCP-інструмент — свій сервер навколо API
  День 18    Планувальник — відкладене/періодичне виконання
  День 19    Композиція — пайплайн з кількох MCP-інструментів
  День 20    Оркестрація — маршрутизація між кількома MCP-серверами

Тиждень 5 (Дні 21-25): RAG (Retrieval-Augmented Generation)
  День 21    Індексація документів — chunking, embeddings, FAISS/SQLite
  День 22    Перший RAG-запит — пошук + LLM, порівняння з/без RAG
  День 23    Реранкінг — фільтрація релевантності, query rewrite
  День 24    Цитати та джерела — антигалюцинації, режим "не знаю"
  День 25    Міні-чат production-like — RAG + пам'ять задачі + джерела

Тиждень 6 (Дні 26-30): Локальні LLM
  День 26    Запуск локальної LLM — Ollama/LM Studio/llama.cpp
  День 27    (завдання не вказано)
  День 28    Локальна LLM + RAG — повністю локальна RAG-система
  День 29    Оптимізація — квантування, параметри, prompt-шаблони
  День 30    Приватний сервіс — LLM як HTTP API на VPS/сервері

Тиждень 7 (Дні 31-35): Прикладні AI-асистенти
  День 31    Асистент розробника — RAG + MCP + git, /help команда
  День 32    (завдання не вказано)
  День 33    Асистент підтримки — RAG + CRM/тікети, контекст користувача
  День 34    (завдання не вказано)
  День 35    (завдання не вказано)


Що отримує учасник на виході
────────────────────────────
- Працюючий AI-агент з пам'яттю, станами та інваріантами
- RAG-система з індексацією, реранкінгом та антигалюцинаціями
- MCP-інтеграції з зовнішніми інструментами та оркестрацією
- Досвід роботи з локальними LLM (розгортання, оптимізація, приватність)
- Прикладні AI-асистенти (розробка, підтримка користувачів)
- Портфоліо з 30+ завдань з кодом і відео
