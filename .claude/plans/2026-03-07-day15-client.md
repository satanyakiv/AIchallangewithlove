# Day 15 Client: Task Lifecycle Screen

## Overview

Day15Screen shows the task lifecycle (Assessment → PlanProposed → Executing → Validating → Completed) alongside the existing session state machine. Key feature: Material3 chips as test data for manual testing of controlled transitions.

## API Contract (server → client)

PsyChatResponse gets 2 new fields (with defaults for backward compat):
- `taskPhase: String = ""` — current task phase name
- `allowedTransitions: List<String> = emptyList()` — what events are valid now

Server route prefix: `/api/agent/psy15/`
